# セキュリティポリシー

## サポート対象バージョン

現在、以下のバージョンに対してセキュリティアップデートを提供しています。

| バージョン | サポート状況  |
| ---------- | ------------- |
| 0.2.x      | ✅ サポート中 |
| < 0.2      | ❌ 非サポート |

## 脆弱性の報告

セキュリティ上の問題を発見した場合は、以下の方法で報告してください。

### 推奨方法: GitHub Security Advisories

1. [Security Advisories](https://github.com/Kou-ISK/sportaglytics/security/advisories) にアクセス
2. "Report a vulnerability" をクリック
3. 脆弱性の詳細を記載して送信

### 代替方法: Issue（非推奨）

**重要**: 公開Issueでのセキュリティ問題の報告は避けてください。セキュリティに関する問題は、上記のSecurity Advisoriesを使用してプライベートに報告してください。

## 報告時に含めるべき情報

脆弱性報告には、以下の情報を含めてください：

- 脆弱性の種類（例: XSS, SQLインジェクション, パストラバーサル等）
- 脆弱性が存在するファイルパスや関連するソースコード
- 再現手順（可能な限り詳細に）
- 影響範囲（どのような被害が想定されるか）
- 修正案（あれば）

## 対応プロセス

1. **受領確認**: 報告受領後、24時間以内に確認の返信を行います
2. **重大度評価**: 7日以内に脆弱性の重大度を評価し、修正計画を策定します
3. **修正とリリース**:
   - Critical（緊急）: 1週間以内にパッチリリース
   - High（高）: 2週間以内にパッチリリース
   - Medium（中）: 1ヶ月以内に通常リリースに含めて対応
   - Low（低）: 次回メジャー/マイナーリリースで対応
4. **公開**: 修正版リリース後、適切なタイミングで脆弱性情報を公開します
