## はじめに

SporTagLytics は、スポーツ映像のタグ付けと分析を効率化するElectronアプリケーションです。

### 主な機能

- **デュアル映像同期**: 寄り/引き2映像の自動同期再生
- **音声同期**: 音響分析による自動オフセット調整
- **イベントタグ付け**: ワンクリックでイベント記録
- **ビジュアルタイムライン**: 波形表示で直感的なイベント編集
- **統計ダッシュボード**: ポゼッション、モメンタム、クロス集計
- **プレイリスト**: フリーズフレーム、描画、クリップ書き出し
- **カスタマイズ可能**: コーディングパネルの自由配置

## クイックスタート

### 1. パッケージ作成

1. メニュー → 「新規パッケージ」
2. パッケージ名とチーム名を入力
3. 保存先ディレクトリを選択
4. 映像ファイルを選択（寄り必須、引き任意）
5. 「作成」ボタンで完了

**パッケージ構造**:

```
PackageName/
├── .metadata/
│   ├── config.json
│   └── sync.json
├── timeline.json
└── videos/
    ├── PackageName 寄り.mp4
    └── PackageName 引き.mp4
```

### 2. 映像再生と基本操作

- **Space**: 再生/一時停止
- **←**: 5秒戻る
- **Shift+←**: 10秒戻る
- **→**: 0.5倍速再生（押下中）
- **シークバー**: クリックで移動

### 3. イベントタグ付け

1. アクションボタンをクリック（開始）
2. もう一度クリック（終了）
3. 自動的にタイムラインに追加

SporTagLyticsには包括的なヘルプウィンドウが組み込まれており、アプリの使い方をいつでも確認できます。

| タイムライン編集       | イベントの編集・移動・複製・プレイリスト追加         |
| 統計ダッシュボード     | ポゼッション・モメンタム・クロス集計の可視化         |
| エクスポート           | タイムラインとクリップの入出力                       |
| プレイリスト           | プレイリスト専用ウィンドウで再生・メモ・描画         |

**ヒント**: 初めて使う機能がある場合は、まずヘルプウィンドウで該当セクションを確認することをおすすめします。

## 映像再生

### デュアル映像同期

- **自動同期**: 音声波形解析で自動同期
- **手動同期モード**: メニュー「同期 > 手動同期モード」（`Cmd+Shift+T`）
- **同期リセット**: メニュー「同期 > 同期オフセットをリセット」（`Cmd+Shift+R`）
- **再解析**: メニュー「同期 > 自動同期（音声解析）」（`Cmd+Shift+S`）

### 再生コントロール

| 操作          | ショートカット       |
| ------------- | -------------------- |
| 再生/一時停止 | `Space`              |
| 5秒戻る       | `←`                  |
| 10秒戻る      | `Shift+←`            |
| 0.5倍速再生   | `→`（押下中）        |
| 2倍速再生     | `Shift+→`（押下中）  |
| 4倍速再生     | `Cmd+→`（押下中）    |
| 6倍速再生     | `Option+→`（押下中） |

※ 倍速再生はキーを離すと通常速度（1.0x）に戻ります。

## イベントタグ付け

### 基本操作

1. **開始**: アクションボタンを1回クリック
2. **終了**: 同じボタンを2回目クリック
3. **保存**: 自動保存

### コーディングパネル編集

**自由配置エディタ（FreeCanvasEditor）**:

- ドラッグ&ドロップで自由配置
- `Shift`/`Cmd`+クリックで複数選択
- グリッドスナップ（10px）
- キャンバスサイズ調整（400-2000px × 300-1500px）

**リンク作成**:

1. 右クリック+ドラッグでリンク作成
2. リンク種別選択:
   - `exclusive`（赤）: 排他的遷移
   - `activate`（緑）: アクティベート
   - `deactivate`（オレンジ）: ディアクティベート
   - `sequence`（青）: シーケンス
3. `Delete` キーで削除

**Undo/Redo**:

- `Cmd+Z`: 元に戻す
- `Cmd+Shift+Z` / `Cmd+Y`: やり直し

### 統計・分析ウィンドウ（`Cmd+Shift+A`）

v0.4.0以降、統計・分析機能は独立したウィンドウで表示されます。

**現在のタブ（v0.5.0）**:

| タブ           | 内容                                                 | バージョン |
| -------------- | ---------------------------------------------------- | ---------- |
| ダッシュボード | カスタマイズ可能なウィジェット配置、テンプレート管理 | v0.4.0     |
| モメンタム     | ポゼッションの流れを棒グラフで可視化                 | v0.2.0     |
| クロス集計     | 任意の軸でヒートマップ表示、ドリルダウン             | v0.2.0     |

**注**: ポゼッション・アクション結果・アクション種別タブはv0.4.0でダッシュボード機能に統合されました。これらの統計情報はダッシュボードのウィジェットまたはカスタムチャート機能で利用できます。

### クロス集計

**カスタマイズ可能な軸**:

- 行軸/列軸: アクション名、結果、種別、修飾子
- フィルタ: チーム別、アクション別
- 画面表示では文字サイズをコンパクト化し、同時に見える情報量を増やしています
- 2段ヘッダ時のスクロール固定を調整し、「合計」列だけ縦長に見える崩れを抑制しています

### ダッシュボード（v0.4.0以降）

**機能**:

- カスタマイズ可能なウィジェット配置
- デフォルトテンプレート（.stadパッケージ形式）
- シリーズ比較機能
- カスタムチャート作成
- 編集モード中はヘッダ操作を `チャート追加 / キャンセル / 保存` に集約し、管理操作とダッシュボード切替は一時的に無効化されます

### モメンタム

- 0基準線が画面中央に見えるよう、Y軸幅を考慮した左右余白で表示します
- 画面表示の目盛りは最小構成（`-max / 0 / +max`）にし、詳細はツールチップで確認します

### AI分析（v0.5.0以降）

**概要**:

AI分析機能は、ローカルLLM（llama.cpp）を使用して映像を自動分析し、質問に対する回答や推奨クリップを生成します。

詳細なセットアップとトラブルシュートは [AI Analysis and Local LLM Setup](ai-analysis.md) を参照してください。データ境界は [Privacy and Data Handling](privacy-and-data-handling.md) にまとめています。

- 右カラムは `根拠 / イベントインサイト（統計） / AI設定` のAccordion構成です（初期表示は根拠のみ展開）
- 根拠リストは初期表示を上位12件に制御し、必要に応じて全件展開できます
- 右カラム専用スクロールを廃止し、タブ全体で一貫したスクロール動作になります

**必要なセットアップ**:

1. **llama.cppバイナリ**:
   - **macOS**: 自動同梱（追加作業不要）
   - **Windows/Linux**: 手動配置が必要

2. **GGUFモデルファイル** (`.gguf`拡張子必須):
   - 配置場所: `public/llama/models/` ディレクトリ
   - 開発中: `public/llama/models/*.gguf`
   - パッケージ版: アプリリソース内の `llama/models/*.gguf`

**llama.cppバイナリの配置方法**:

アプリは以下の優先順位でバイナリを検索します：

1. **環境変数** (最優先):
   - `SPORTAGLYTICS_LLAMA_PATH`: SporTagLytics専用パス
   - `LLAMA_CPP_PATH`: 汎用llama.cppパス
   - 例: `export SPORTAGLYTICS_LLAMA_PATH="/usr/local/bin/llama-cli"`

2. **アプリ内蔵パス**:
   - パッケージ版: `<app>/Contents/Resources/llama/<platform>/`
   - 開発版: `public/llama/<platform>/`
   - `<platform>` = `darwin` (macOS) / `win32` (Windows) / `linux`

3. **バイナリ名の候補** (優先順):
   - `llama-completion`
   - `llama-cli`
   - `llama`
   - `main`
   - Windows: 上記に `.exe` 拡張子

**使い方**:

1. 統計・分析ウィンドウを開く（`Cmd+Shift+A`）
2. 「AI分析」タブを選択
3. テンプレートから質問を選択するか、自由にテキスト入力
4. 「実行」ボタンでAI分析を開始
5. 結果として以下が表示されます：
   - **要約**: 質問に対する簡潔な回答
   - **仮説**: データに基づく推論
   - **根拠**: タイムライン上の関連イベント（ハイライト表示）
   - **推奨クリップ**: AI が選んだ重要シーン

**プレイリスト作成**:

- 「プレイリスト作成」ボタンで、推奨クリップから自動的にプレイリストを生成できます
- 生成されたプレイリストは新しいウィンドウで開きます

**AI分析の仕組み**:

- **ハイブリッド根拠検索**: テキスト、ラベル、メモ、時間、レアラベルを組み合わせて関連イベントを抽出
- **グラウンディング**: タイムラインデータを基にした事実ベースの回答生成
- **ローカル処理**: 全てのデータはローカルで処理され、外部に送信されません
- **ストリーミング進捗**: 出力文字数とチャンクごとの進捗がリアルタイムで表示
- **タイムアウト制御**: デフォルト120秒（2分）、実行中のキャンセルも可能

**制限事項**:

- v0.5.0では macOS のみサポート（Windows/Linux版llama.cppバイナリは未同梱）
- モデルファイルは `.gguf` 拡張子で `public/llama/models/` に配置が必要
- 環境変数 `SPORTAGLYTICS_LLAMA_PATH` または `LLAMA_CPP_PATH` でバイナリパスをカスタマイズ可能

### 分析結果の出力

分析ウィンドウの export 操作から以下を出力できます。

| 出力                   | 内容                                                   |
| ---------------------- | ------------------------------------------------------ |
| 構造化サマリーをコピー | 現在の分析ビューの要約をクリップボードへコピー         |
| PNG保存                | 現在の分析ビューを画像として保存。長い内容は分割保存可 |
| 分析レポートPDF保存    | ダッシュボード、モメンタム、クロス集計をPDF化          |

詳細は [Analysis Report Export](analysis-report.md) を参照してください。

## プレイリスト機能

### プレイリスト作成

#### タイムラインからの追加（v0.2.0以降）

1. タイムラインからイベントを選択（複数選択可）
2. 右クリック → 「プレイリストに追加」
3. 開いている全てのプレイリストウィンドウに追加されます  
   （ウィンドウが開いていない場合は新規ウィンドウが自動で開きます）

- ツールバーのプレイリストボタンからも同じ操作が可能です

#### AI分析からの追加（v0.5.0以降）

1. 統計・分析ウィンドウで「AI分析」タブを開く
2. 質問を入力し「実行」ボタンで分析
3. 「プレイリスト作成」ボタンで推奨クリップからプレイリストを自動生成
4. 新しいプレイリストウィンドウが自動的に開きます

**注意**: v0.5.0でプレイリスト作成方法が統一され、両方の方法とも同じElectron APIを使用するようになりました。

#### 複数ウィンドウ管理

- 同時に複数のプレイリストウィンドウを開くことができます
- 新しいウィンドウは自動的にカスケード表示されます（前のウィンドウから50pxずつオフセット）
- 同じ`.stpl`ファイルを複数回開こうとした場合は、既存のウィンドウにフォーカスします

### プレイリストファイル（.stpl）形式

プレイリストは `.stpl` 拡張子のパッケージ形式で保存されます。

**パッケージ構造**:

```
MyPlaylist.stpl/              # パッケージディレクトリ（Finderでは単一ファイルに見える）
├── playlist.json             # プレイリストメタデータ（テキストファイル）
└── videos/                   # 映像ファイル（埋め込み形式のみ）
    ├── パス_01JGXXX123ABC/   # <アクション名>_<itemId>
    │   ├── angle1.mp4        # プライマリ映像
    │   └── angle2.mp4        # セカンダリ映像（存在する場合）
    ├── シュート_01JGXXX456DEF/
    │   ├── angle1.mp4
    │   └── angle2.mp4
    └── ...
```

**形式の種類**:

- **埋め込み形式（embedded / スタンドアロン）**: 映像ファイルをFFmpegで切り出して`videos/`に保存、パッケージ単体で再生可能（大容量）
- **参照形式（reference）**: 外部映像ファイルへのパスを保存、軽量（元の映像ファイルが必要）
  - 元映像が移動/削除された場合、参照形式のプレイリストは再生できません

**保存時の選択**:

- 初めてプレイリストを保存する際、保存ボタンを押すと形式選択ダイアログが表示されます
- 既に保存されているプレイリストは、その形式で上書き保存されます
- 埋め込み形式では、クリップの開始時刻は`0秒`起点に変換されます（切り出し動画のため）

**ファイルの閲覧**:

1. Finderでプレイリストファイルを右クリック
2. 「パッケージの内容を表示」を選択
3. `playlist.json` をテキストエディタで開いて内容確認・編集可能

**未保存変更の管理**:

- ウィンドウタイトルに未保存変更がある場合は `*` マークが表示されます
- ウィンドウを閉じる際、未保存の変更がある場合は確認ダイアログが表示されます
  - 「保存」: 変更を保存してから閉じる
  - 「保存しない」: 変更を破棄して閉じる
  - 「キャンセル」: 閉じずにウィンドウを維持
- 保存した描画・フリーズ設定はプレイリスト読み込み時に復元されます

### プレイリストウィンドウ

**再生機能**:

- 連続再生
- ループ再生
- フリーズフレーム
- アングル表示切替（アングル1/アングル2/デュアル）

**フリーズの挙動**:

- 描画のタイムスタンプに到達すると自動停止
- 再開は再生ボタン/`Space`で手動

**コントロール表示**:

- コントロールは映像エリアにマウスをホバーした時のみ表示されます
- コントロールボタンとスライダーは小型化され、より邪魔にならないデザインになっています

**描画機能**:

- ペン、選択（ドラッグ移動・Delete削除）、矩形、円、線、矢印、テキスト
- 色選択、線幅調整
- Undo、全クリア
- 描画は「作成した時刻」に紐づき、再生中は該当時刻付近で表示されます
- デュアル表示中は描画対象（メイン/サブ）を切り替え可能

**メモ/選択**:

- チェックボックスで複数選択し、`Backspace`で一括削除
- メモはプレイリスト内のみに保存され、タイムラインには反映されません

### クリップ書き出し

**書き出しモード**:

- 1ファイル: 全クリップを結合
- インスタンスごと: イベント単位
- アクションごと: アクション種別単位

**書き出し範囲**:

- 全体
- 選択中アイテムのみ

**オーバーレイ**:

- アクション名、通番、ラベル、メモ
- 描画（アノテーション）とフリーズフレームを含めた書き出しに対応

**アングル**:

- 全アングル（アングルごとに個別書き出し）
- 単一アングル
- マルチアングル（2映像を並列）

**進捗表示**:

- 書き出し中はSnackbarで進捗が表示されます

## データのインポート・エクスポート

関連 ADR: [0009 Timeline Import/Export Interoperability](adr/0009-timeline-import-export-interoperability.md)

### エクスポート

**形式**:

- JSON形式
- CSV形式
- SCTimeline形式（Sportscode互換）

**項目**:

- イベントID、アクション名、開始/終了時刻、ラベル、メモ

### インポート

- JSON形式のタイムライン
- SCTimeline形式のタイムライン
- インポート時は現在のタイムラインを置き換えます（マージは行われません）

## 設定

### 全般設定

- テーマ（ライト/ダーク/システム）
- クリップ書き出しオーバーレイ（表示/非表示、表示項目の切替）

### コーディングパネル設定

- レイアウト（デフォルト/カスタム）
- ボタン/ラベルの配置編集（ドラッグ&ドロップ）
- ボタンサイズ、色、テキスト、ホットキーの調整
- グリッドスナップ、キャンバスサイズ
- レイアウトの作成/複製/削除、インポート/エクスポート

### ホットキー設定

- カスタムホットキー登録
- 競合チェック
- デフォルトに戻す
- システム標準のショートカット（Cmd+Q等）は割り当て不可

## プライバシーとデータ

SporTagLytics は local-first の desktop app です。映像、タイムライン、プレイリスト、AI 分析プロンプトはローカルで処理され、現行実装では cloud sync、telemetry、remote analytics、cloud LLM provider は使いません。

保存場所、recent package、localStorage、AI 分析時の一時ファイル、issue 報告時の注意点は [Privacy and Data Handling](privacy-and-data-handling.md) を参照してください。

## トラブルシューティング

### 音声同期がうまくいかない

1. 両映像に音声が含まれているか確認
2. 手動オフセット調整を試す
3. `.metadata/sync.json` を削除して再計算

### 映像が再生されない

1. ファイル形式を確認（MP4/MOV推奨）
2. コーデックを確認（H.264推奨）
3. ファイルパスに日本語が含まれていないか確認

### タイムラインが保存されない

1. ディレクトリの書き込み権限を確認
2. ディスク容量を確認
3. `timeline.json` が破損している場合は削除して再作成
